## 2024-05-24 - DoS vulnerability via unlimited file downloads
**Vulnerability:** The `src/html2md/cli.py` script called `requests.get()` without `stream=True` and without checking `Content-Length`. This would allow an attacker to trigger the conversion of an arbitrarily large file (e.g. 10GB or an infinite stream), exhausting the application's memory and causing an Out of Memory (OOM) crash.
**Learning:** `requests.get` by default downloads the entire response body into memory. Since this application accepts user-provided URLs to download and parse, it is highly susceptible to DoS attacks via malicious endpoints. We must be very careful to always bound memory usage. Also, mocking `builtins.open` globally in tests can break internal Python mechanisms (like `gettext` attempting to load translation files during `argparse` initialization).
**Prevention:** Always use `stream=True` with `requests.get` when downloading data from untrusted sources. Check the `Content-Length` header (if available) to fail early, and read the content using `iter_content(chunk_size)` while maintaining a running total of bytes received. Enforce a hard maximum limit (e.g. 20MB). Furthermore, when mocking `builtins.open` in tests, scope the mock using a `side_effect` function that only intercepts target files and forwards all other calls to the original `open`.
