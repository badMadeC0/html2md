## 2024-05-24 - Path Traversal bypass using os.path.commonpath
**Vulnerability:** The output path containment check used `os.path.commonpath([real_outdir, real_out_path]) != real_outdir` to prevent path traversal. However, if paths are not properly trailed with slashes, `os.path.commonpath` might return a prefix that matches `real_outdir` for sibling directories (e.g. `['/tmp/out', '/tmp/outside/a.md']` returns `/tmp`, but wait - earlier it was assumed `commonpath` returns prefix strings. Actually `commonpath` is component-based, but using `pathlib`'s `.relative_to()` provides a much more robust and standard way to check path containment and avoids edge cases with trailing slashes or symlink resolution ordering).
**Learning:** `os.path.commonpath` can be tricky to use correctly for security checks involving containment. `pathlib.Path.relative_to()` explicitly checks if a path is a subpath of another, throwing a `ValueError` if it is not.
**Prevention:** In the future, explicitly use `pathlib.Path().resolve().relative_to()` when validating that a user-provided file path falls within a designated base directory.
