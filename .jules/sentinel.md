## 2024-05-24 - Server-Side Request Forgery (SSRF) in URL Fetcher
**Vulnerability:** The CLI tool accepted arbitrary URLs to fetch and convert to Markdown without validating if the resolved IP address belonged to an internal, private, loopback, or link-local network. This allowed Server-Side Request Forgery (SSRF) where the tool could be used to probe internal networks or access sensitive internal services.
**Learning:** Checking the URL scheme (e.g. `http` or `https`) is insufficient to prevent SSRF. Attackers can use public domain names that resolve to private IPs (like `127.0.0.1` or `169.254.169.254`) to bypass simple scheme or hostname string checks.
**Prevention:** Before making a network request to an external URL, always resolve the hostname and verify that the resulting IP address is public using an IP validation library (e.g., Python's `ipaddress` module `ip.is_private`, `ip.is_loopback`, etc.). Note that complete SSRF prevention requires pinning the resolved IP or disabling redirects.
