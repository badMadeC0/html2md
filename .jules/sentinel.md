## 2024-05-10 - Server-Side Request Forgery (SSRF) Mitigated in html2md CLI
**Vulnerability:** The `html2md` CLI accepted any HTTP/HTTPS URL and fetched it using `requests.get()` without validating the resolved IP address. This allowed attackers to provide URLs pointing to internal networks, loopback addresses, or cloud metadata endpoints, exposing sensitive internal services.
**Learning:** `requests.get()` follows redirects automatically, meaning an attacker could provide a public URL that redirects to a malicious internal IP (DNS rebinding / redirect SSRF bypass). Manual redirection handling is necessary to ensure the IP is validated before each request.
**Prevention:** Always validate the resolved IP address of a URL before making a request. Use `socket.gethostbyname` and `ipaddress` properties (`is_private`, `is_loopback`, `is_link_local`, `is_multicast`) to block access to non-public networks. When fetching untrusted URLs, set `allow_redirects=False` and manually process redirects to ensure every resolved IP is checked in the chain.
