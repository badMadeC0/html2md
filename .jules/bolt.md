## 2024-05-05 - Fast-pathing string operations in loops
**Learning:** In tight loops like CSV generation, string methods like `.lstrip()` are incredibly expensive because they allocate new strings (except in some CPython fast paths where no leading whitespace is removed, but the method call overhead remains). `json.loads` (without custom hooks) guarantees exact base types (like `str` and `dict`), so using `type(x) is str` is significantly faster than `isinstance(x, str)` and perfectly safe.
**Action:** When working in hot loops (like data parsing/exporting), cache attribute lookups (`get = dict.get`), avoid unnecessary allocations by adding fast-path checks (e.g., check `char.isspace()` before calling `.lstrip()`), and prefer exact type checks over `isinstance` for known JSON objects.
