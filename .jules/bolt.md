## 2024-05-24 - Optimizing CSV export performance with plain writer vs DictWriter
**Learning:** When using `csv.DictWriter` to filter and map fields from a parsed JSON dictionary, allocating a new dictionary per row for the `DictWriter` is computationally expensive due to the overhead of dictionary creation and the DictWriter's internal validation. While mutating the parsed dictionary in place could save allocations, it introduces risks of key collisions if the output mapping overwrites fields that need to be read in later iterations.
**Action:** The safest and most performant approach is to bypass `csv.DictWriter` entirely. Instead, use a plain `csv.writer`, pre-extract the list of input keys, and use a list comprehension (`w.writerow([_sanitize_value(rec.get(name, "")) for name in input_names])`) to directly extract the needed values from the parsed JSON dictionary. This avoids per-row dictionary allocations and `DictWriter` overhead while maintaining correctness.
