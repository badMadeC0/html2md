## 2024-05-13 - Optimize CSV formula sanitization
**Learning:** Found a hot loop in `html2md/log_export.py` where `lstrip()` was called on nearly every string when checking for dangerous prefixes. Avoid string allocation overhead by adding a fast path for strings that do not start with spaces `not value[0].isspace()`. Swapped tuple for string (`"=+-@"`) for slightly faster `in` lookups.
**Action:** When performing string checks inside tight loops over large datasets, look for ways to bypass allocating string methods (like `lstrip()`) by using fast index lookups and character class checks (`isspace()`).
