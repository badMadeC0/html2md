## 2024-05-18 - Avoid over-optimization anti-patterns
**Learning:** Using `type(x) is dict` instead of `isinstance(x, dict)` is a recognized Python anti-pattern that breaks inheritance and duck-typing, even if it is technically faster and valid in some micro-contexts (like parsing exact JSON dicts). Speed optimizations should not sacrifice idiomatic Python readability and robustness.
**Action:** Stick to `isinstance` for type checking unless absolutely necessary and documented, focusing optimizations on clear algorithmic or computational bottlenecks instead (like avoiding unnecessary string allocations).
